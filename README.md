# 🚀 Avito Classifieds Dashboard

Современный личный кабинет продавца для управления объявлениями с интегрированным AI-ассистентом на базе Ollama.

## 🛠 Технологический стек

-   **Frontend**: React + Vite (на порту `80` через Nginx)
-   **Backend**:Node.js + Fastify (на порту `8080`)
-   **AI Engine**: Ollama (модель `llama3`)
-   **Infrastructure**: Docker + Docker Compose

---

## 🐳 Запуск через Docker

Приложение полностью контейнеризировано. Один запуск разворачивает всю необходимую инфраструктуру, включая автоматическую загрузку AI-модели.

### Требования
-   Установленный [Docker](https://www.docker.com/get-started)
-   Docker Compose

### Быстрый старт
Выполните команду в корневой директории проекта:

```bash
docker compose up --build
```

После завершения сборки приложение будет доступно по адресу:
👉 **[http://localhost](http://localhost)**

## 🏗 Архитектура контейнеров

В составе Docker-окружения работают 4 взаимосвязанных сервиса:

1.  **`frontend`**: Основной интерфейс приложения. 
    -   Использует **Nginx** как веб-сервер и **Reverse Proxy**.
    -   Все запросы к API (`/api/`) и Ollama (`/ollama/`) проходят через этот контейнер, что исключает проблемы с CORS.
2.  **`backend`**: Серверная часть на Fastify.
    -   Обрабатывает CRUD операции с объявлениями.
    -   Слушает внутренний порт `8080`.
3.  **`ollama`**: Локальный движок для запуска LLM.
    -   Обеспечивает работу AI-ассистента (генерация описаний, оценка цен).
4.  **`ollama-setup`**: Вспомогательный сервис.
    -   Автоматически скачивает модель `llama3` при первом запуске, если она еще не установлена.

## 🧠 Принятые решения

1.  **Работа с изображениями**: 
    -   Поскольку текущий бэкенд не поддерживает загрузку и хранение медиафайлов, для демонстрации объявлений используются внешние плейсхолдеры (`picsum.photos`). 
    -   Ссылки на изображения генерируются динамически на основе `id` объявления.
2.  **Собственная библиотека UI**: 
    -   Для проекта был разработан собственный набор UI-компонентов (кнопки, поля ввода, модальные окна, уведомления). 
    -   Это позволило реализовать уникальный дизайн "Business Luxury" без ограничений готовых UI-фреймворков.
3.  **Unit-тестирование**: 
    -   Для обеспечения надежности ключевые функции и компоненты были покрыты юнит-тестами с использованием **Vitest**. 
    -   Тесты проверяют логику фильтрации, валидацию и работу AI-интеграций.

## 💡 Особенности реализации

-   **Единая точка входа**: Благодаря Nginx проксированию, вам не нужно беспокоиться о портах `8080` или `11434` в браузере. Всё работает через стандартный порт `80`.
-   **Автоматизация AI**: Модель `llama3` (~4.7 ГБ) скачивается автоматически при первом запуске. Вы можете следить за прогрессом в логах контейнера `ollama-setup`.
-   **Кросс-браузерность**: Сборка фронтенда оптимизирована для работы в большинстве современных браузеров.
