# Avito Dashboard

Современный личный кабинет продавца для управления объявлениями с интегрированным AI-ассистентом на базе Ollama.

## 🛠 Технологический стек

-   **Frontend**: React + Vite (на порту `80` через Nginx)
-   **Backend**:Node.js + Fastify (на порту `8080`)
-   **AI Engine**: Ollama (модель `llama3`)
-   **Infrastructure**: Docker + Docker Compose

---

## 🐳 Запуск через Docker

Приложение полностью контейнеризировано. Один запуск разворачивает всю необходимую инфраструктуру, включая автоматическую загрузку AI-модели.

### Требования
-   Установленный [Docker](https://www.docker.com/get-started)
-   Docker Compose

### Быстрый старт
Выполните команду в корневой директории проекта:

```bash
cd avito
docker compose up --build
```

После завершения сборки приложение будет доступно по адресу:
👉 **[http://localhost](http://localhost)**

---

## 🏗 Архитектура контейнеров

В составе Docker-окружения работают 4 взаимосвязанных сервиса:

1.  **`frontend`**: Основной интерфейс приложения. 
    -   Использует **Nginx** как веб-сервер и **Reverse Proxy**.
    -   Все запросы к API (`/api/`) и Ollama (`/ollama/`) проходят через этот контейнер, что исключает проблемы с CORS.
2.  **`backend`**: Серверная часть на Fastify.
    -   Обрабатывает CRUD операции с объявлениями.
    -   Слушает внутренний порт `8080`.
3.  **`ollama`**: Локальный движок для запуска LLM.
    -   Обеспечивает работу AI-ассистента (генерация описаний, оценка цен).
4.  **`ollama-setup`**: Вспомогательный сервис.
    -   Автоматически скачивает модель `llama3` при первом запуске, если она еще не установлена.

## 🧠 Принятые решения

1.  **Работа с изображениями**: 
    -   Поскольку текущий бэкенд не поддерживает загрузку и хранение медиафайлов, для демонстрации объявлений используются внешние плейсхолдеры (`loremflickr`). 
    -   Ссылки на изображения генерируются динамически на основе `id` объявления.
2.  **Собственная библиотека UI**: 
    -   Для проекта был разработан собственный набор UI-компонентов (кнопки, поля ввода, модальные окна, уведомления). 
3.  **Unit-тестирование**: 
    -   Для обеспечения надежности ключевые функции и компоненты были покрыты юнит-тестами с использованием **Vitest**. 
    -   Тесты проверяют логику фильтрации, валидацию и работу AI-интеграций.

## 🚀 Дополнительные возможности

Помимо базового CRUD-функционала, в проекте реализованы продвинутые инструменты для продавцов:

1.  **Интерактивный AI-чат**: 
    -   Полноценный боковой чат (Sidebar), доступный на странице редактирования.
    -   Ассистент адаптируется под контекст вашего объявления (название, категория, параметры).
    -   Возможность вести живой диалог для тонкой настройки описания.
2.  **Визуальное сравнение (Diff Viewer)**:
    -   При генерации нового описания открывается инструмент сравнения.
    -   Вы наглядно видите, что именно изменил искусственный интеллект (стилистика, ключевые слова, структура) перед тем, как применить правки.
3.  **Премиальная тёмная тема**:
    -   Элегантная палитра для комфортной работы в ночное время.
    -   Умное сохранение выбора в `localStorage`.

## 💡 Особенности реализации

-   **Единая точка входа**: Благодаря Nginx проксированию, вам не нужно беспокоиться о портах `8080` или `11434` в браузере. Всё работает через стандартный порт `80`.
-   **Автоматизация AI**: Модель `llama3` (~3.0 ГБ) скачивается автоматически при первом запуске. Вы можете следить за прогрессом в логах контейнера `ollama-setup`.

## 🖥 Локальный запуск (без Docker)

Если вы хотите запустить компоненты отдельно для разработки:

### 1. AI Engine (Ollama)
1. Установите **Ollama** с официального сайта [ollama.com](https://ollama.com/).
2. Запустите загрузку и выполнение модели:
   ```bash
   ollama run llama3
   ```
3. Убедитесь, что сервер Ollama доступен по адресу `http://localhost:11434`.

### 2. Backend
1. Перейдите в директорию сервера: `cd server`.
2. Установите зависимости: `npm install`.
3. Запустите сервер:
   ```bash
   npm start
   ```
   Сервер будет доступен по адресу `http://localhost:8080`.

### 3. Frontend
1. Перейдите в директорию клиента: `cd client`.
2. Установите зависимости: `npm install`.
3. Запустите приложение:
   ```bash
   npm run dev
   ```
   Интерфейс будет доступен по адресу `http://localhost:5173`. 
   
> [!NOTE]
> Локальный запуск фронтенда настроен на проксирование запросов `/ollama` -> `http://localhost:11434`. Для работы с API бэкенда убедитесь, что переменная `VITE_API_BASE_URL` указывает на `http://localhost:8080/api`.
